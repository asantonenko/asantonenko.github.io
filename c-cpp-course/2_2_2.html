Ідентифікаторами зазвичай є слова чи фрази англійською. Дуже бажано використати осмислені ідентифікатори. Якщо ідентифікатор повинен складатися з декількох слів, розділити пробілами їх не можна. Бажано використовувати один із наступних стилів іменування:

  • under_score (через_підкреслення) – ідентифікатори пишуться малими літерами, слова поділяються підкресленням – цей стиль часто використовується у мові C++, зокрема для імен зі стандартної бібліотеки – наприклад: unordered_map – невпорядкований асоціативний масив, forward_list – однонаправлений список або out_of_range – виняткова ситуація, що сигналізує про вихід за межі допустимого діапазону;
  • ALL_CAPS_UNDER_SCORE – теж саме, лише великими літерами, слова розділяються підкресленням – в C і C ++ зазвичай використовується для імен констант, наприклад INT_MAX – максимальне число, що міститься в типі int, INT_MIN – мінімальне число, що міститься в типі int;
  • camelCase або lowerCamelCase (верблюжийРегістр)  – кожне слово, крім першого, пишеться з великої літери, в C# і Java застосовується для імен змінних, а в Java і для імен методів, в C++ теж можна використовувати для імен змінних і/або функцій;
  • PascalCase або  UpperCamelCase – кожне слово пишеться з великої літери, як випливає з назви, використовується в мові Pascal, але також і в мовах C#/Java для імен класів, а також C# для імен методів, в C++ теж часто використовується для імен користувацьких (не стандартних) класів, а також іноді і для імен функцій і методів.

Як Ви бачите, на жаль, у C++ немає єдиного стандарту іменування елементів мови, на відміну від мов Java та C#, наприклад. А в мові C, як більш старій і архаїчній, взагалі для імен функцій стандартної бібліотеки найчастіше використовуються аббревіатури, що складно розшифровуються, такі наприклад, як назва функції isalnum, яка перевіряє символи на буквено-цифрові, замість нормального (нескороченого) імені is_alpha_numeric або isAlphaNumberic.

Примітка: Досить зрозуміло, чому деякі назви функцій у мові Сі нагадують якісь шифрування або абревіатури. Справа в тому, що, по перше, у 70-х роках, коли створювалася мова Сі, програма вся вводилася символ за символом з клавіатури (тоді як сучасні середовища пропонують підказки з автодоповненням), крім того, мінімізація розміру вихідного коду була важливішою, ніж зараз (через те, що апаратні ресурси, насамперед пам'ять, комп'ютерів на той час були дуже обмежені). У сучасних програмах використання скорочених імен виду RRect або delusr замість нормальних повних імен RoundedRectangle (прямокутники із заокругленими кутами) або delete_user (видалити користувача) вкрай небажано.

Отже, хоча для мов C/C++ немає єдиного універсального стилю іменування, проте бажано вибрати деякий стандарт іменування для свого проекту і суворо дотримуватися його. Якщо ж Ви працюєте в деякій компанії, то у неї цілком може бути корпоративний стандарт, як, наприклад, Google C++ Style Guide -> Naming. У цих правилах, наприклад, описано використання стилю under_score для імен локальних змінних та стилю PascalCase для імен типів, методів та функцій.

Приклади ідентифікаторів.

  • Хорошими ідентифікаторами є: 
    Circle, scale, index, isDivisible, hasChildren, table_name, AddTableEntry, DeleteUrl, UrlTableProperties (останні чотири приклади взяті з Google C++ Style Guide -> Naming)
  • Дані ідентифікатори підходять для тимчасових об'єктів, малої області дії:
    tmp, i, j, x, y, z, a, b, c, d, x0, x1, x2, y0, y1, y2
  • Погані ідентифікатори:
    krug, InDeX, addtableentry, I, l, O, o  (останні чотири – це І велике англійське, Л маленьке англійське, О велике і о маленьке – ніколи не використовуйте такі ідентифікатори для уникнення плутанини – подивіться як вони виглядають у звичайному шрифті –  I, l, O, o)
  • Не можна використовувати у якості ідентифікаторів:
    table name, ping-pong, 2power10, now!, return
