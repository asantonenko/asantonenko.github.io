Контейнер set — множина.

Синтаксис:

#include <set>

set<тип_елемента> ім'я_множини;

Згадаймо математичне визначення множини. З точки зору математики, множина — це набір елементів, причому порядок елементів у множині не має значення, а один і той самий елемент не може повторюватися декілька разів.

Таким чином, у математичній нотації \(\left \{ 1, 2, 3\right\} = \left \{ 3, 2, 1\right\}\) і \(\left \{ 1, 1, 2, 2, 2\right\} = \left \{ 1, 2\right\}\), тоді як \(\left \{ 1, 2, 3\right\} \neq \left \{ 1, 2, 4\right\}\).

Нагадаю, що в мові C++ існують асоціативні контейнери, порядок елементів у яких визначається внутрішніми властивостями цих елементів, а не порядком їх додавання до контейнера. Саме до таких контейнерів і належить контейнер set (множина). Цей контейнер містить впорядкований набір унікальних елементів. Після додавання певної кількості елементів, вони завжди будуть впорядковані за зростанням. Якщо спробувати додати вже існуючий у контейнері елемент, помилка не виникне, але елемент не буде доданий. Звісно, щоб цей контейнер працював, має бути реалізовано порівняння елементів базового типу контейнера — за замовчуванням використовується операція < для порівняння, але можна використати й іншу функцію-компаратор або функціональний об'єкт-компаратор.

За замовчуванням створюється порожня множина, наприклад,

set<int> primes;

куди можна додавати елементи за допомогою операції insert, наприклад:

primes.insert(2);
primes.insert(5);
primes.insert(7);
primes.insert(2);
primes.insert(3);

Хоча ми додали елементи не по порядку, і двійку додали двічі, все одно отримаємо впорядковану множину з чотирьох елементів \(\left \{ 2, 3, 5, 7\right\}\). Проте, якщо ми хочемо отримати таку множину, набагато зручніше записати це через універсальну ініціалізацію (починаючи з версії C++11):

set<int> primes = {2, 3, 5, 7};
