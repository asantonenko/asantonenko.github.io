Итак, операции декремента и, особенно, инкремента используются весьма часто, но обычно они используются отдельно, а не в сложных выражениях, и в данном случае не важно какую форму использовать - префиксную или постфиксную. Например, при поиске количества элементов, удовлетворяющих какому-то условию, найдя элемент, мы увеличиваем на 1 его количество. 

Например, мы хотим найти количество положительных элементов массива arr размера size. Мы можем это сделать в следующем фрагменте:

int count = 0;
for (int i = 0; i < size; i++)
{
    if (arr[i] > 0)
    {
        count++;
    }
}
cout <<"Количество положительных элементов в массиве равно " <<count <<endl;

Изначально полагаем количество элементов равно 0 (мы еще не нашли ни одного положительного элемента). Потом используем переменную i чтобы перебрать все номера элементов от 0 до size - 1 (см. урок Операторы цикла). При этом после каждой итерации цикла происходит увеличение переменной i на 1, как раз при помощи операции инкремента i++. Далее мы указываем, если элемент массива arr с номером i больше нуля, то увеличим значение переменной count на 1 (с помощи операции инкремента count++). Подробнее про массивы см. модуль "Массивы. Контейнеры языка C++".

В данной программе используются постфиксные операции инкремента - но это просто сделано по традиции, ничем не хуже будет использовать префиксные операции, т.к. результат операции игнорируется, то разницы нет. Т.е. фрагмент программы можно записать в виде: 

int count = 0;
for (int i = 0; i < size; ++i)
{
    if (arr[i] > 0)
    {
        ++count;
    }
}
cout <<"Количество положительных элементов в массиве равно " <<count <<endl;

Давайте приведем пример, где уместно использовать операцию инкремента в более сложном выражении. Пусть у нас есть переменная index, задающая позицию (индекс) в массиве arr, куда нужно записать число number. Это действие можно записать так:

arr[index] = number;

Но предположим, что следующее число будет нужно записать в следующую позицию, с индексом на единицу больше. Можно, конечно, написать два оператора:

arr[index] = number;
index++;

Но при таком подходе можно забыть написать увеличение переменной index. Так как в данном случае логика следующая: записываем элемент по определенному индексу в массив и сразу, обязательно, увеличиваем индекс, то вполне логично записать один комбинированный оператор: 

arr[index++] = number;

И вот здесь уже существенная разница между префиксной и постфиксной формой операторов - в данном случае использование префиксной формы: 

arr[++index] = number;

приведет совсем к другому эффекту: вначале увеличим index, и затем по этому увеличенному индексу запишем элемент массива, т.е. это эквивалентно такой последовательности операторов:  

index++;
arr[index] = number;
