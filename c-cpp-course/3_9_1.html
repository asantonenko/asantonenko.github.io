Обратите внимание на некоторую тонкость: пусть есть некоторая функция f и задано выражение: f(x,y). Как его трактовать? Это функция с двумя параметрами или это функция с одним параметром x,y - полученным как результат операции «запятая»? В таких случаях всегда трактуется однозначно - запятая в скобках после имени функции всегда обозначает разделитель параметров, а не операцию «запятая». Т.е. f(x,y) - это вызов функции с двумя параметрами.

Операция «запятая» используется обычно тогда, когда по синтаксису необходимо использовать одно выражение, а необходимо записать два и более выражений. 

Пример: В цикле

for (i = 0, j = n - 1; i < j; i++, j--)
{
    ...
}

используются сразу две управляющие переменные. До первой точки с запятой в круглых скобках расположено одно выражение - инициализация цикла, но мы по сути вписали применение двух операций присваивания, условие продолжения - пока i < j, а в итерации - выражении после второй точки с запятой - описывается изменения сразу двух переменных - увеличении переменной i и уменьшения переменной j - нам опять понадобилась операция «запятая», чтобы записать два выражения в одном.  Таким образом, в цикле переменная i будет увеличиваться, а переменная j будет уменьшаться, пока они не встретятся в середине. Подробнее о циклах, см. урок "Операторы цикла".

Операцией «запятая» не нужно злоупотреблять. Если необходимо несколько действий внутри оператора if, while, do while или for используйте фигурные скобки, а не операцию «запятая». 

Т.е. вместо 

if (b < 0) a=-a, b=-b;

пишите 

if (b < 0) {a=-a; b=-b;}

или еще лучше 

if (b < 0) 
{
   a = -a; 
   b = -b;
}

Подробнее о том, зачем здесь нужны фигурные скобки, см. шаг "Составной оператор" урока "Структурное программирование", а также урок "Операторы ветвления".
