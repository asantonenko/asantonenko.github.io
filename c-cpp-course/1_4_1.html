Починаючи з другого модуля, ми будемо систематично розглядати основні аспекти програмування мовами C/C++. У даному уроці розглянуті якісь найпростіші, але хоча б мінімально корисні програми можна зробити мовою C++. При цьому мовні засоби докладно не пояснюються, а лише вказується, навіщо вони потрібні і що вони роблять.

Отже, найперші комп'ютери в історії людства були дуже великими і дорогими, але мали дуже скромні можливості, та й зручності роботи з ними вони не могли забезпечити. Питання – навіщо ж тоді вони були потрібні, що вони вміли робити? Вони вміли робити математичні обчислення за формулами, закладеними в них програмістами, і хоча робили це повільніше за будь-який сучасний мобільний телефон, але все ж таки набагато швидше за людину. Пройшло багато часу з тих пір – але все ж, програмісту найлегше показати користь комп'ютерних програм, запрограмувавши якісь обчислення. Інша річ, що тепер обчислювати за математичними формулами вміє навіть пошуковий рядок google, а в будь-якому смартфоні є вбудований калькулятор. Забудемо поки що про це і спробуємо створити власний калькулятор мовою C++.

Отже, що ж має зробити програма калькулятор? Вона повинна отримати вихідні дані від користувача, провести необхідні обчислення і вивести на екран результат. Більше того, більшість програм, які ми писатимемо в цьому курсі, будуть працювати саме в такій послідовності. Отже, нам потрібно навчитися реалізовувати три етапи:

  1. вводити дані з клавіатури;
  2. здійснювати розрахунки за деякими формулами;
  3. виводити результати на екран.

Але спочатку необхідно оголосити дані, з якими ми збираємося працювати – так звані змінні. І тут з'являється місце для рішення програміста – з даними якого типу повинна працювати програма? Так наша програма є калькулятором – то очевидно, що програма буде працювати з числами. З точки зору мови C++ основний вибір тут такий: чи з цілими, чи з дійсними числами буде працювати наша програма. Так як при роботі калькулятора зазвичай можна задавати як цілі, так і дробові (нецілі) числа, ми будемо використовувати дійсний тип double. Якщо точніше, то це дійсний тип з подвійною точністю. Отже, найперша заготівля виглядатиме так:

int main()        //основна функція програми
{                 //початок основного блоку
    double x, y;  //дві дійсні змінні

    //Тут буде решта програми

    return 0;     //повідомляємо, що програма успішно завершила роботу
}                 //кінець основного блоку

Нагадую, що текст, що починається з двох слешів представляє коментар і потрібен лише для пояснення роботи програми, компілятор C++ його ігнорує.

Ми оголосили дві змінні x та y, які може задавати користувач. Давайте введемо їх із клавіатури:

#include <iostream>  //заголовний файл для введення та виведення

using namespace std; //використовуємо простір імен std

int main()        
{                 
    double x, y;  

    cout << "Введіть два числа через пробіл:" << endl; //виводимо підказку для користувача
    cin >> x >> y;                                     //вводимо значення змінних з клавіатури

    //Тут буде решта програми

    return 0;     
}

Я вказую коментарі тільки для нових рядків коду, тому Вам легко відстежити, що з'явилося нового. Окрім власне введення змінних із клавіатури (або точніше зі стандартного потоку введення), я також попередньо роздрукував інструкцію для користувача програми, щоб він розумів, що потрібно робити. Це дуже бажано, якщо використовується інтерактивний діалоговий режим роботи з користувачем. Тепер обчислимо суму, різницю, добуток та частку двох введених чисел x та y.

#include <iostream>  

using namespace std; 

int main()        
{                 
    double x, y;  

    cout << "Введіть два числа через пробіл:" << endl; 
    cin >> x >> y;          
                           
    double sum        = x + y; //розрахунок суми
    double difference = x - y; //розрахунок різниці
    double product    = x * y; //розрахунок добутку
    double quotient   = x / y; //розрахунок частки

    //Тут буде решта програми

    return 0;     
}

Зверніть увагу, я використовую англійські назви (імена) для змінних, при цьому я оголошую і відразу обчислюю змінну за деякою формулою. Всі змінні, звичайно ж дійсного (double) типу. Я вставив зайві пропуски, щоб гарно вирівняти формули, це робити було не обов'язково (але, як мені здається, в такому вигляді код виглядає краще). Чого ще не вистачає? Правильно, роздруківка результатів:

#include <iostream>  

using namespace std; 

int main()        
{                 
    double x, y;  

    cout << "Введіть два числа через пробіл:" << endl; 
    cin >> x >> y;               
                      
    double sum        = x + y; 
    double difference = x - y; 
    double product    = x * y; 
    double quotient   = x / y; 

    cout << x << " + " << y << " = " << sum        << endl; //друкуємо суму
    cout << x << " - " << y << " = " << difference << endl; //друкуємо різницю
    cout << x << " * " << y << " = " << product    << endl; //друкуємо добуток
    cout << x << " / " << y << " = " << quotient   << endl; //друкуємо частку

    return 0;     
}

Зверніть увагу на те, як ми друкуємо результати. У рядку cout << x << " + " << y << " = " << sum << endl; послідовно виводяться у стандартний вивід такі значення: значення змінної x, знак + з двома пробілами до і після нього, значення змінної y, знак = з двома пробілами до і після нього, значення змінної sum і, нарешті, виведення переводиться на наступний рядок за допомогою маніпулятора endl. Також зверніть увагу, що на друк виводяться тільки ті пробіли, що знаходяться в лапках, а пробіли поза лапками розставлені для зручності читання програми, для цього ж я вирівняв у стовпчик записи <<endl; в кінці рядків. 

Перевірити роботу цієї програми можна за адресою: https://onlinegdb.com/22BNGXFu1 

Примітка. Якщо Ви захочете запустити подібний код у певному середовищі під ОС Windows, наприклад Visual Studio або Code::Blocks, то Вам може знадобитися деяка конструкція на початку main, наприклад, setlocale, щоб українські літери коректно відображалися (наприклад, в такий спосіб - https://onlinegdb.com/7qVJh4Uhc). В цілому, проблема коректної обробки українських чи російських літер в консолі Windows на мові С++ несподівано дуже складна, відсилаю тих, хто цікавиться сюди.
