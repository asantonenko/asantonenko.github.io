Взаимоисключающие условия

Очень часто в программе приходится использовать взаимоисключающие условия. Рассмотрим пример решения квадратного уравнения. После вычисления дискриминанта у нас есть три варианта: дискриминант больше 0 (два корня), дискриминант равен 0 (один корень) и дискриминант меньше 0 (нет корней). Эти условия взаимоисключающие, то есть мы вначале проверяем первое условие, если оно не выполнено, проверяем второе и т.д. Таким образом, каждый следующий if оказывается вложенным в предыдущий.

Если попробовать записать общую структуру вложенных операторов if строго по правилам, расставляя все фигурные скобки и отступы, получиться так:

double D = b * b - 4 * a * c;

if (D > 0)
{
	cout <<"Два корня!" <<endl;
}
else
{
	if (D == 0)
	{
		cout <<"Один корень!" <<endl;
	}
	else 
	{
		if (D < 0)
		{
			cout <<"Корней нет!" <<endl;
		}
	}
}

Такая структура фигурных скобок и отступов очень громоздка, мы получаем лесенку из операторов со всё большими отступами. Обычно вместо этого условно объединяют два ключевых слова else и if в одну конструкцию else if, не ставя данных необязательных скобок. Кроме того, последний оператор if вообще можно не записывать, так как если не выполнены первые два условия, то автоматически выполнено третье. Получаем следующую структуру:

double D = b * b - 4 * a * c;

if (D > 0)
{
	cout <<"Два корня!" <<endl;
}
else if (D == 0)
{
	cout <<"Один корень!" <<endl;
}
else 
{
	cout <<"Корней нет!" <<endl;
}

Данный вариант выглядит намного компактнее и проще предыдущего. 

Приведем также полный текст программы решения квадратного уравнения:

#include <iostream>
#include <cmath>

using namespace std;

int main() 
{
	double a, b, c; //коэффициенты квадратного уравнения
	
	cout <<"Введите три коэффициента уравнения (a b c):" <<endl;
	
	cin  >>a >>b >>c;
	
	cout <<"Уравнение имеет вид: " <<a <<"*x*x + " <<b<<"*x + " <<c <<" = 0" <<endl;
	
	double D = b*b - 4*a*c; //Вычисляем дискриминант
	
	if (a==0) //Это вообще не квадратное уравнение
	{
		cout <<"Данное уравнение не является квадратным! "<<endl;
	}
	else if (D>0) //Если дискриминант больше нуля, то есть два различных вещественных корня 
	{
		double x1 = (-b+sqrt(D)) / (2*a);
		double x2 = (-b-sqrt(D)) / (2*a);
		cout <<"Уравнение имеет два различных вещественных корня: " <<endl;
		cout <<"x1= " <<x1 <<"  x2= " <<x2 <<endl;
	}
	else if (D==0)
	{
		double x12 = -b/(2*a);
		cout <<"Уравнение имеет один вещественный корень: " <<endl;
		cout <<"x12= " <<x12 <<endl;
	}
	else
	{
		cout <<"У уравнения нет вещественных решений!" <<endl;
	}
	
	return 0;
}

Обратите внимание в каких местах объявлены переменные в программе. Напомним, что в языке С++ используются следующие правила хорошего стиля:

  1. Переменные нужно делать настолько локальными, насколько это возможно (без потери смысла или эффективности работы). Например, переменные x1 и x2 объявлены только внутри оператора if (D>0), поэтому они не существуют в других ветках или вне if.
  2. Переменные желательно объявлять по мере необходимости - как можно ближе к месту использования. Напомним, что в старых версиях языка Си (С89) переменные нужно было объявлять вначале блока. Поэтому у тех, кто переходит со старых версий языка C, или языка Pascal, или кого обучили по старинке, может появиться весьма вредная привычка объявлять все переменные вначале. В современном C++ это совершенно не нужно, более того, это считается плохим стилем! Поэтому вначале мы объявили только переменные a, b, c для исходных данных - коэффициентов квадратного уравнения, а вот переменную D для дискриминанта объявили только после ввода значений переменных a, b и c.
  3. Желательно, по возможности, объявляя переменную, сразу же присваивать ей нужное значение. В программе так происходит с переменными D, x1, x2, x12. Только переменным a, b, c не присваивается начальное значение - но только лишь потому, что они считываются с клавиатуры и это действие нельзя (без дополнительных усилий - создания вспомогательных функций или использования сложных конструкций) записать в виде инициализации переменной.
