Нульовий вказівник

Окрім вказівників, які вказують на корисні дані, існує поняття нульового вказівника (null pointer). Такий вказівник нікуди не вказує. Іншими словами, він сигналізує про відсутність корисних даних. Для цього зарезервовано спеціальну адресу з числовим значенням нуль. Гарантується, що за цією адресою не може бути корисних даних.

Розіменування нульового вказівника є операцією з невизначеною поведінкою. Якщо програма аварійно завершується з помилкою (що найімовірніше), це ще найкращий сценарій. У загальному випадку це може призвести до непередбачуваних наслідків.

У мові Сі нульовий вказівник позначається макросом NULL, значення якого дорівнює (void*) 0. У Сі це коректно, оскільки нетипізований вказівник можна неявно перетворити на будь-який тип:

int * ptr = NULL;

У мові C++ такий підхід неможливий — неявне перетворення void* на типізований вказівник заборонене. Тому автори C++ вирішили позначати нульовий вказівник цілим числом 0. Макрос NULL у C++ також має значення 0. Однак використання числа для позначення вказівника може призводити до плутанини, особливо при роботі з шаблонними функціями.

У C++11 цю проблему вирішено введенням ключового слова nullptr, яке однозначно позначає нульовий вказівник:

int * pi = nullptr;
double * pd = nullptr;
char * pc = nullptr;
string * ps = nullptr;
