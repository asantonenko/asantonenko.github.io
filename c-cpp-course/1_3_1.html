За традицією, почнемо з програми мовою програмування С, яка виведе рядок Привіт, світе! (англ. Hello world!, тим більше, що ця традиція започаткована у класичній книзі «Мова програмування Сі» Браяна Кернігана та Денніса Рітчі):

/* Програма Привіт, світе! */
#include <stdio.h>

int main()
{
    printf("Привіт, світе!\n");
    return 0;
}

Перший рядок цієї програми зовсім не обов'язковий — це коментар, який не впливає на виконання програми, але підказує програмісту (але не користувачеві), що програма робить. Коментар починається на символи /* і закінчується */. Компілятор мови C просто ігнорує коментар.

Будь-який текст у подвійних лапках позначатиме рядкову константу, а якщо ми хочемо використовувати подвійну лапку в рядковій константі, то перед нею потрібно поставити знак «зворотний сліш». Цей знак означає, що наступний символ має особливе значення, зокрема \n означає символ переходу на новий рядок. По суті такий символ ми набираємо на клавіатурі, коли натискаємо клавішу Enter. Заодно, зазначимо, що \n - це один символ, а не два.

Примітка: Після кожного виведеного на консоль рядка прийнято переводити курсор на наступний рядок. Навіть прийнято переводити курсор на наступний рядок після останнього рядку виводу, або як у нашому випадку, після єдиного рядка, що виводиться. Так прийнято робити, але можна цього і не зробити і написати:

/* Програма Привіт, світе! */
#include <stdio.h>

int main()
{
    printf("Привіт, світе!");
    return 0;
}

Єдиною відмінністю цієї програми є те, що вона не переведе курсор виведення на новий рядок, після того як надрукує повідомлення "Привіт, світе!".

Зауважимо, що наприкінці кожного оператора стоїть крапка з комою. Вона вказує на кінець оператора, таким чином різні оператори можна записувати як в одному рядку, так і різних, головне щоб в кінці кожного була крапка з комою. Зауважимо також, що після закриваючої фігурної дужки крапка з комою не ставиться (за рідкісним винятком, який буде обговорено надалі). Отже програму можна написати коротше:

#include <stdio.h>
int main()
{
    printf("Привіт, світе!\n"); return 0;
}

Інша справа, що ця економія навряд чи виправдана тут, все ж таки логічніше в цьому випадку писати кожен оператор в окремому рядку. Більш того, порожній рядок перед функцією main візуально відокремлює її від попередніх конструкцій, а коментар на самому початку програми часто говорить про її назву, автора та/або призначення.

Зауважимо, що якщо Ви використовуєте інтегроване середовище розробки в операційній системі Microsoft Windows, то, написавши подібну програму Ви можете побачити краказябри замість українських букв. У такому разі Вам доведеться привітати світ рядком "Hello, world!\n" або навіть "Pryvit, svite!\n".

Щоб вивести два рядки, можна використовувати як два виклики функції printf, так і один виклик, у якому в рядку буде два символи \n. Ці дві програми призводять до одного результату:

#include <stdio.h>

int main()
{
    printf("Перший рядок!\n");
    printf("Другий рядок!\n");   
    return 0;
}

та 

#include <stdio.h>

int main()
{
    printf("Перший рядок!\nДругий рядок!\n");     
    return 0;
}

Те, що надрукує будь-яка з цих програм, буде виглядати в консолі так:

Перший рядок!
Другий рядок!

Якщо пропустити \n у першому рядку, то обидва рядки будуть надруковані в одному:

#include <stdio.h>

int main()
{
    printf("Перший рядок!"); 
    printf("Другий рядок!\n");     
    return 0;
}

Результат: 

Перший рядок!Другий рядок!

Якщо пропустити \n тільки в другому рядку, то візуально результат не зміниться

#include <stdio.h>

int main()
{
    printf("Перший рядок!\n"); 
    printf("Другий рядок!");     
    return 0;
}

 Результат: 

Перший рядок!
Другий рядок!

Хіба що наступне виведення (вже інших програм) може злитися з рядком Другий рядок! - тому і все-таки прийнято вставляти символ переведення рядка навіть після виведення останнього рядка.
