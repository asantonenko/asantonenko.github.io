Давайте тепер напишемо цей приклад мовою програмування С++. Зауважу, що оскільки мова C++ в основному сумісна з мовою C, минулий приклад теж є абсолютно коректною програмою мовою C++, але все ж таки нам цікаво познайомиться зі специфічними саме для C++ конструкціями. Отже, наша програма виглядатиме так:

// Програма Привіт, світе!
#include <iostream>

int main()
{
    std::cout << "Привіт, світе!" << std::endl;
    return 0;
}

Перший рядок — це коментар у стилі C++, втім, починаючи зі стандарту C99, так можна писати і мовою С. Компілятор ігнорує будь-який текст, починаючи з двох слешів // і до кінця рядка. І тим більше, мовою C++ можна писати коментарі за допомогою /* та */. Підключення заголовного файлу iostream - рядок #include <iostream> — потрібно, щоб працювали конструкції std::cout и std::endl. Знову ж таки зауважимо, що для стандартних заголовних файлів C++  правильне ім'я саме iostream, а не iostream.h

Операція << в даному виразі означає операцію виведення, або, якщо точніше, операцію вставки в потік. std::cout - це стандартний потік виведення (в стилі C++), за замовчуванням цей потік позначає виведення на екран (скоріше за все, чорне консольне вікно). А std::endl - це маніпулятор, виведення якого в потік призводить до переведення виводу на новий рядок (аналогічно виводу символу '\n', але також додатково відбувається виштовхування буфера потоку, цей момент нехай Вас поки що не хвилює). Використовувати цей маніпулятор не обов'язково, запис  std::cout << "Привіт, мир!\n"; теж цілком коректна і призведе до схожого ефекту.

Ми будемо використовувати також конструкцію using namespace std; дозволяє вкоротити подальшу частину програми, а саме не писати найменування простору імен std:: перед стандартними конструкціями. Тоді оновлена програма виглядатиме так:

// Програма Привіт, світе!
#include <iostream>

using namespace std;

int main()
{
    cout << "Привіт, світе!" << endl;
    return 0;
}

Необходимо заметить, что хотя данная строка  using namespace std; и удобна для новичков в программировании, ее использование в профессиональных программах не является хорошей практикой и часто не оправдано. Тем не менее, я буду использовать данную конструкцию в своих примерах, и, более того, далее я буду именовать поток std::cout и манипулятор std::endl по их сокращенным именам cout и endl .

Если Вы хотите напечатать две строки, то Вам нужно два раза использовать endl — в конце каждой из них (напомню, что принято переводить строку после последней строки вывода). Например, 

#include <iostream>

using namespace std;

int main()
{
    cout << "Это первая строка!" << endl;
    cout << "А это вторая строка!" << endl;
    return 0;
}

Впрочем, это можно вывести и один раз указывая cout:

#include <iostream>

using namespace std;

int main()
{
    cout << "Это первая строка!"   << endl
         << "А это вторая строка!" << endl;
    return 0;
}

Обратите внимание, что в первой строке нет точки с запятой - значит оператор ещё не закончен. Символы операции << выровнены пробелами, так чтобы находились друг под другом исключительно для красоты (стиля кодирования), работать будет и без этого. Вообще эту конструкцию можно было записать в одну строку:

cout << "Это первая строка!" << endl << "А это вторая строка!" << endl;

Напечатано в стандартный поток вывода (на экран) будет две строки - благодаря манипуляторам endl.
